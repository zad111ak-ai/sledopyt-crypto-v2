# ПОСТ 5: Что-то сломалось? Как починить установку с помощью бесплатных нейросетей

---

**🔧 Серия: "Локальный AI-агент своими руками" — Пост 5/6**

При установке что-то пойдёт не так — это закон подлости. Но у вас есть бесплатный техподдержчик в браузере.

---

**📐 Универсальный алгоритм:**

1. Сделайте скриншот ошибки: `Win + Shift + S`
2. Откройте в браузере бесплатную нейросеть:
   • **chat.deepseek.com** — отличный для кода
   • **gemini.google.com** — умеет анализировать скриншоты
   • **claude.ai** — в бесплатном тарифе тоже ок
3. Загрузите скриншот и напишите:

*"Я устанавливаю Hermes Agent на WSL 2 с Ubuntu 26.04. На шаге [описание] появилась эта ошибка. Объясни причину и дай пошаговое решение. Команды пиши с учётом что я новичок."*

4. Следуйте инструкциям нейросети

---

**🔴 Типовые проблемы и быстрые решения:**

**«Virtualization disabled in BIOS»**
→ Вернитесь в Пост 1 и включите VT-x / AMD-V

**«wsl --install падает с ошибкой 0x800...»**
→ Обновите Windows до последней версии. Иногда нужен патч

**«hermes: command not found»**
→ Выполните `source ~/.bashrc` или перезапустите Ubuntu

**«API ключ не принимается»**
→ Проверьте, что добавили его в `~/.bashrc` без пробелов вокруг `=`

**MiMo Code не копирует текст**
→ Используйте `Ctrl+Shift+C/V` или правую кнопку мыши

**Ubuntu не запускается**
→ `wsl --shutdown` → затем снова `wsl`

---

**💡 Главный принцип:**

Не стесняйтесь задавать нейросетям "глупые" вопросы. Прикладывайте скриншоты. Спрашивайте "куда нажимать" и "что это за кнопка".

Через 2-3 итерации проблема решается в 99% случаев.
